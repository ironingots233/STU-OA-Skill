
# STU-OA.skill 

> *「懒惰是人类进步的阶梯」*

这是一个为 AI Agent 设计的定制化技能（Skill）套件，旨在自动化抓取、解析并高度概括汕头大学（STU）办公自动化（OA）系统的通知。

通过结合 Python 数据抓取工具与严格的 Agent 提示词指令（Prompt），该套件能够根据用户的自然语言查询（如“今日OA日报”、“本周有什么比赛”），精准提取信息并生成无冗余、格式化的 OA 通知汇报。可无缝集成到支持 Function Calling / Tool Calling 的大语言模型框架中（如 Coze, Dify, OpenAI Assistants 等）。



## ✨ 核心特性

* 📅 **动态时间过滤**：通过推算日期，精准抓取过去 N 天内的有效通知。
* 🧹 **深度内容清洗**：深度去除 HTML 标签、脚本、样式表以及 Word 文档粘贴特有的冗余元数据，还原纯净正文。
* 🤖 **标准化函数调用**：提供标准的 `functions.json` 规范，Agent 易于理解和调用。
* 🎯 **严格的模板输出**：内置核心指令文件，强制 Agent 根据意图（如单日、多日、特定主题、日报总览）采用 4 种特定的精简输出模板，绝对禁止冗长的废话和寒暄。

## 📁 目录结构与说明

本项目包含以下四个核心文件，分别对应 Agent 的“大脑”、“双手”与“使用说明”：

* **`SKILL.md`**：**Agent 的系统提示词 (System Prompt)**。定义了 Agent 的人设、行为逻辑、调用策略（A/B/C/D 种情况）以及绝对禁令（禁止废话、禁止思考外显）。
* **`functions.json`**：**工具的 API 声明**。以标准的 JSON Schema 描述了两个 Python 工具的入参和出参，供 LLM 识别和调用。
* **`get_oa_news.py`**：**工具一（目录抓取）**。接收时间参数（`days_ago`），爬取汕大 OA 系统的通知列表，并为每条通知分配临时 ID。
* **`stu_oa_scraper.py`**：**工具二（正文抓取）**。接收工具一生成的 ID 数组（`item_ids`），精准抓取对应通知的网页内容并进行深度文本清洗，返回纯文本数据。

## ⚙️ 核心工作流

1. **意图识别**：用户发送请求（例：“生成今日OA日报”或“最近有讲座吗”）。
2. **列表抓取**：Agent 提取时间参数，调用 `get_oa_news` 获取近期通知的列表（包含标题、日期和 ID）。
3. **详情抓取**：如果用户的意图需要了解详情（如提取日报的核心行动项、或查询讲座的具体时间地点），Agent 会提取相关通知的 ID，静默调用 `stu_oa_scraper` 获取正文内容。
4. **格式化输出**：Agent 根据 `SKILL.md` 中的强指令约束，将正文内容高度概括为一句话，并匹配相应的模板（模板A/B/C/D）进行极简输出。

## 🚀 部署与使用指南

### 1. 依赖安装
对于运行 Python 工具的环境，只需确保安装了 Python 3.x 标准库即可，本项目所有工具代码均使用 Python 内置库（`urllib`, `re`, `html`, `json`, `datetime`）编写，**无需安装第三方依赖**。

### 2. 在 Agent 平台集成
1. **创建工具/插件**：在您的 Agent 平台（如 Coze/Dify）中创建两个自定义工具。
2. **配置代码与 Schema**：
   * 将 `get_oa_news.py` 和 `stu_oa_scraper.py` 的代码分别填入工具执行代码块。
   * 根据 `functions.json` 中的定义，配置这两个工具的参数描述（入参类型、必填项等）。
3. **设置人设与回复逻辑**：
   * 将 `SKILL.md` 中的所有内容复制，粘贴到 Agent 的 **“人设与回复逻辑” (System Prompt / Instructions)** 中。
4. **绑定工具**：将刚才创建的两个 OA 工具绑定到该 Agent 上。

## ⚠️ 注意事项

* **网络环境**：由于抓取的是 `oa.stu.edu.cn`，请确保运行 Python 脚本的服务器/环境具备访问汕头大学外网或校园网的权限。若 OA 系统设置了严格的 IP 限制或验证码拦截，爬虫可能会失效。
* **隐私与权限**：本项目当前实现基于公开免登录可访问的 `list.jsp` 列表页。如涉及需内网统一身份认证拦截的页面，需后续扩展 Cookie/Token 模拟登录功能。
* **模板约束**：LLM 偶尔会出现“幻觉”或忘记指令的情况，如果发现其开始寒暄（例如说“好的，我为您查询...”），请调低模型温度（Temperature = 0.1 ~ 0.3）以增强其对 `SKILL.md` 绝对禁令的服从性。
